# 多卡联合支付最优组合方案说明书

## 📋 系统概述

**多卡联合支付**是一种智能化的会员卡组合使用方案，当单张会员卡余额不足或为了获得更优惠的组合折扣时，系统自动计算最优的多卡使用策略，最大化客户的优惠权益。

## ⚙️ 核心算法规则

### 1. noDiscount服务优先处理 🚫
- **不打折服务**：标记为 `noDiscount: true` 的服务项目按原价收费
- **可打折服务**：标记为 `noDiscount: false` 的服务项目享受会员卡折扣
- **混合支付**：先支付不打折服务，再支付可打折服务
- **安全保障**：任何会员卡都无法对不打折服务应用折扣

### 2. 优先级策略
- **折扣率优先**：折扣率低的卡片优先使用（6折 > 7折 > 8折）
- **小余额优先**：相同折扣率下，余额较少的卡片优先使用
- **清空策略**：优先清空小额卡，避免余额碎片化

### 3. 计算公式

**不打折服务支付：**
```
卡片扣款 = min(剩余不打折金额, 卡片剩余余额)
```

**可打折服务支付：**
```
卡片最大覆盖原价 = 卡片余额 ÷ 折扣率
实际覆盖原价 = min(卡片最大覆盖原价, 剩余可打折金额)
卡片扣款金额 = 实际覆盖原价 × 折扣率
优惠金额 = 实际覆盖原价 - 卡片扣款金额
```

### 4. 精度处理
- **四舍五入**：所有金额计算结果精确到分（小数点后2位）
- **分步计算**：每张卡独立计算，避免累积误差
- **Decimal.js**：使用高精度数学库确保货币计算准确

## 💡 典型应用场景

### 场景一：单个noDiscount服务 🚫

**服务项目：**
- 剪发+修面（优惠套餐）：¥58，noDiscount=true

**会员卡：**
- 6折自定义卡：余额¥100

**系统处理：**
- 不打折服务：¥58（按原价收费）
- 卡片扣款：¥58
- 折扣金额：¥0
- **结果**：总计支付¥58，无任何折扣

**关键特点：** ✅ 确保不打折服务不受任何卡片折扣影响

### 场景二：混合服务支付（noDiscount + 可打折）

**服务项目：**
- 剪发+修面（优惠套餐）：¥58，noDiscount=true
- 纯中草药染发‧短发：¥158，noDiscount=false

**会员卡：**
- 6折卡：余额¥100
- 8折卡：余额¥150

**系统计算过程：**

**第1步 - 6折卡处理**
- 支付不打折服务：¥58（按原价扣除）
- 剩余余额：¥100 - ¥58 = ¥42
- 支付可打折服务：¥42 ÷ 0.6 = ¥70原价，扣¥42
- 6折卡总扣：¥100，给予折扣：¥70 - ¥42 = ¥28

**第2步 - 8折卡处理**
- 剩余可打折服务：¥158 - ¥70 = ¥88原价
- 扣款：¥88 × 0.8 = ¥70.4
- 8折卡给予折扣：¥88 - ¥70.4 = ¥17.6

**最终结果：**
- 总金额：¥216.00
- 实际支付：¥170.40
- 总折扣：¥45.60
- **验证**：✅ noDiscount服务¥58按原价，可打折服务享受优惠

### 场景三：充足余额的最优组合

**客户情况：**

- 优惠金额：¥25 - ¥20 = ¥5
- 剩余待支付：¥50 - ¥25 = ¥25

**第2步 - 500元储值卡（处理剩余金额）**
- 最大覆盖原价：¥300 ÷ 0.7 = ¥428.57
- 实际覆盖原价：¥25（剩余待支付金额）
- 扣卡金额：¥25 × 0.7 = ¥17.5
- 优惠金额：¥25 - ¥17.5 = ¥7.5

**最终结果：**
- 原价：¥50.00
- 实际支付：¥37.50
- 总优惠：¥12.50
- 平均折扣：7.5折

## 🔍 常见问题

**Q: 为什么要优先使用小余额卡？**
A: 避免小额余额长期闲置，提高卡片利用率，减少客户管理多张卡的复杂度。

**Q: 折扣计算是否会有误差？**
A: 系统采用四舍五入到分的精确计算，每步都有明确的数学公式，确保计算准确。

**Q: 如果所有卡片余额都不够怎么办？**
A: 系统会最大化使用所有可用卡片，剩余部分提示客户用现金补足。

**Q: 可以手动选择使用哪张卡吗？**
A: 系统提供最优方案，如需特殊需求可联系店员进行手动调整。

**Q: noDiscount服务为什么不能打折？**
A: 这是商家设定的特殊服务政策，通常用于套餐优惠或特价服务，确保价格公平性。

**Q: 混合支付时如何确保计算正确？**
A: 系统采用先处理不打折服务，再处理可打折服务的两阶段算法，每一步都有严格的数学验证。
